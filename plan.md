> A clean, fast, developer-focused tab session manager for Chrome.

## Phase 1 ‚Äî Research & Plan (this document)

### What I learned from `a11y-annotator` (the reference extension)
- **Manifest V3**: `manifest_version: 3`, service worker background (`type: module`),
  `action.default_popup`, `commands` block for keyboard shortcuts, `icons` at 16/48/128.
- **Service worker pattern**: `chrome.runtime.onMessage` listener that returns `true`
  for async responses; `chrome.commands.onCommand` for keyboard shortcuts.
- **Icons**: generated programmatically with Pillow (`generate_icons.py`) ‚Üí PNG at 16/48/128.
- **Packaging**: a `.zip` of the extension files for the Chrome Web Store.
- **Monetization**: a `stripe.json` holding the payment link; a `landing/index.html`.
- **Docs**: `store-listing.md`, `launch-plan.md`, `test-results.md`, privacy + README.

### Manifest V3 essentials for TabForge
- Service workers are **ephemeral** ‚Äî no long-lived globals. All state must live in
  `chrome.storage`. Alarms (`chrome.alarms`) survive worker restarts; `setInterval` does not.
  ‚Üí Auto-save MUST use `chrome.alarms`, not `setInterval`.
- Popups are short-lived documents ‚Äî logic runs fresh each open. No inline scripts
  (CSP) ‚Üí all JS in external files, all event handlers via `addEventListener`.
- Permissions needed:
  - `tabs` ‚Äî read tab url/title to save a session, create tabs to restore.
  - `storage` ‚Äî persist sessions (sync for cross-device, local for large data).
  - `alarms` ‚Äî periodic auto-save.
  - `commands` ‚Äî keyboard shortcuts (Ctrl+Shift+S save, Ctrl+Shift+R restore last).
  - No `host_permissions` needed: `chrome.tabs` url/title are available with `tabs`.

### Storage strategy
- `chrome.storage.sync` has hard limits: **100 KB total, 8 KB per item, 512 items**.
  A session of many tabs can exceed 8 KB/item. ‚Üí Strategy:
  - Store the **session index** (ids, names, counts, timestamps) in `sync` so the list
    syncs across devices and is tiny.
  - Store **full session payloads** (the tab arrays) in `local` keyed by id, which has
    a much larger quota (~5 MB, or unlimited with `unlimitedStorage`).
  - On a sync-quota failure, gracefully fall back to `local` for the index too.
- This keeps the door open for the Pro "cloud sync" feature without re-architecting.

### Freemium gating (structure now, enforce cleanly)
- A single `pro` flag in storage + a `LIMITS` table in `utils.js`.
- Free tier: up to 5 saved sessions, auto-save interval ‚â• 15 min.
- Pro tier: unlimited sessions, auto-save down to 1 min, plus placeholders wired for
  cloud sync / tab groups / duplicate detection / templates (feature-flagged, not stubbed UI).
- Gating is centralized in `utils.canSaveMore()` / `utils.minAutoSaveMinutes()` so the
  paywall is one code path, not scattered checks.

### Module wiring
- `storage.js` and `utils.js` are **ES modules** (`export`), imported by:
  - `background.js` (declared `"type": "module"` in manifest).
  - `popup.js` (loaded as `<script type="module">`).
- Shared logic lives in the modules; `popup.js` and `background.js` stay thin controllers.

### Data model
```js
// Session (payload, stored in local under key `session:<id>`)
{ id, name, createdAt, updatedAt, auto: bool, tabs: [{ url, title, pinned, favIconUrl }] }

// Index entry (stored in sync under key `index`)
{ id, name, createdAt, updatedAt, auto: bool, tabCount }

// Settings (stored in sync under key `settings`)
{ autoSave: bool, intervalMinutes: number, pro: bool, theme: 'dark'|'light' }
```

## Key risks & mitigations
| Risk | Mitigation |
|------|-----------|
| Service worker killed mid auto-save | Use `chrome.alarms`; reload state from storage each fire |
| `storage.sync` 8KB/item limit | Split: index‚Üísync, payloads‚Üílocal |
| Restoring 50 tabs hangs the browser | Batch tab creation; restore into a new window |
| chrome:// / extension tabs can't be reopened | Filter unsupported schemes on save, warn on restore |
| Lost data on bad import | Validate schema before writing; never overwrite, always merge/add |
